# Architecture

## Overview

The opencode_api package provides a type-safe Dart client for the opencode.ai API using the retrofit code generator.

## Project Structure

```
lib/
├── opencode_api.dart              # Main library exports
└── src/
    ├── api/
    │   ├── opencode_client.dart    # Retrofit @RestApi definitions
    │   ├── opencode_client.g.dart # Generated code
    │   └── opencode_client_extension.dart  # Convenience methods
    ├── models/
    │   ├── project.dart            # Project model
    │   ├── session.dart            # Session model
    │   ├── message.dart            # Message model
    │   ├── file_node.dart          # FileNode model
    │   └── response.dart          # API response models
    ├── exceptions/
    │   └── opencode_exception.dart # Custom exception
    └── utils/
        └── logger.dart             # Platform-aware logger
```

## Design Decisions

### Retrofit for API Client

We use retrofit to generate type-safe HTTP client code. The `@RestApi` annotation defines the base URL, and `@GET`, `@POST`, etc. define endpoints.

### Generated vs Hand-written Code

- **Generated** (`*.g.dart`): Code generated by retrofit and json_serializable - do not edit manually
- **Hand-written**: Models, client definitions, extensions, exceptions

### Platform-aware Logger

The package uses conditional exports to provide different logger implementations:

- `logger_native.dart` - Full logger for desktop/mobile (uses dart:io)
- `logger_web.dart` - No-op stub for web platforms
- `logger.dart` - Exports appropriate implementation based on platform

### Error Handling

- `OpencodeException` wraps all API errors
- `userMessage` property provides safe messages without implementation leaks
- Detailed errors logged via logger for debugging

### Null Safety

All model fields are nullable (`String?`, `int?`, etc.) because the opencode API may return null for missing fields.

## Dependency Flow

```
User Code
    │
    ▼
OpencodeClient (Retrofit generated)
    │
    ▼
Dio (HTTP client)
    │
    ▼
opencode.ai HTTP API
```

## Extension Methods

The `OpencodeClientExtension` provides convenience methods that:
- Handle `HttpResponse<T>` wrapper extraction
- Add try-catch with logging
- Return typed data directly

Example:
```dart
// Raw method returns HttpResponse<HealthResponse>
final raw = await client.getHealthRaw();

// Extension method returns HealthResponse directly
final health = await client.getHealth();
```
